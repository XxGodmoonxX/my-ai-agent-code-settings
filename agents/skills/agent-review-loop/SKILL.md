# レビュー＆自動修正ループ

`external-agent-review` スキルを使ってレビューを取得し、High/Medium 指摘を修正します。
このサイクルを **2 回** 繰り返してください。

## サイクルの手順

### ステップ 1: レビューを取得

`/external-agent-review` スキルを実行してレビュー結果を取得する。

### ステップ 2: High / Medium 指摘の抽出

レビュー結果から `High` または `Medium` とラベル付けされた指摘のみを抜き出す。
`Low` の指摘はこのサイクルでは無視する。

### ステップ 3: 指摘の修正

抽出した各 High/Medium 指摘に対して:
1. 対象ファイルと行を確認する
2. Edit ツールで修正する
3. 修正内容を簡潔に記録する（例: `src/foo.ts:42 — null チェック追加`）

High/Medium が 1 件もない場合は修正をスキップして次のサイクルへ進む。

---

## ループ制御

- 1 サイクル目完了後、**必ず** 2 サイクル目を実行する（早期終了なし）
- サイクル間でコミットは挟まない。2 サイクル目では 1 サイクル目の修正も含めた差分がレビューされる
- 2 サイクル目完了後、最終サマリを報告して終了:

```
【サイクル 1】
  High: N 件 → 修正済み
  Medium: N 件 → 修正済み

【サイクル 2】
  High: N 件 → 修正済み
  Medium: N 件 → 修正済み

合計修正件数: N 件
```

## 注意事項

- 修正は High/Medium 指摘の範囲に限定し、それ以外のリファクタリングは行わない
- `external-agent-review` が失敗した場合はエラーを表示してユーザーに確認を求める
