# ユーザーレベルの指示

## ユーザーのスキルセット

バックエンドは初心者。Rails や Go まわりの実装をする際はなぜそうなるのかの理由も合わせて説明する。

## シェルコマンド

不要な `echo`（見出し・区切り・終了コード出力など装飾目的のもの）を付けない。コマンドは単体でそのまま実行する。

コマンド実行時に `cd` を実行したい場合、まず `pwd` して現在いるディレクトリを確認してから実行する。

## git コマンド

`git` コマンドに `-C <dir>` フラグは不要。カレントディレクトリがすでにリポジトリ内にあるので、`git log` などをそのまま使う。

git 管理下にあるファイルやフォルダをリネームしたり移動させる時は `git mv` コマンドを使用する。

PRのタイトルには `feat:` / `fix:` / `docs:` などの Conventional Commits の prefix を付けない。コミットメッセージとPRタイトルは別物で、PRタイトルはレビュアーが内容を把握するためのものなので、人が読んでわかりやすい日本語タイトルにする。

## 言語

プランや回答は常に日本語で出力する。

## Codex レビューの呼び出し

`/codex:review` は `disable-model-invocation: true` のため、モデル（Claude）が Skill ツールで自動呼び出しすることはできない（ユーザーが自分でスラッシュコマンドを打つ場合は動く）。Agent Skill の中から呼ぶ場合も含め、モデル側から Codex レビューを実行したい場合は、その実体である `codex-companion.mjs` を Bash で直接叩く。

パイプ・グロブ・コマンド置換は使わず、2 ステップに分けて実行する（許可ルールにプレフィックス一致させるため）：

1. `ls -t ~/.claude/plugins/cache/openai-codex/codex/` を実行し、出力の 1 行目から最新バージョンのディレクトリ名を読み取る
2. そのバージョンを埋め込んで実行する（パスは `~/` 始まり・クォートなし）：

```bash
node ~/.claude/plugins/cache/openai-codex/codex/<バージョン>/scripts/codex-companion.mjs review --wait
```
