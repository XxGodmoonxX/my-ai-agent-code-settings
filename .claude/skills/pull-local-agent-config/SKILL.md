# ローカルのユーザー設定をリポジトリにpullする

PC上（ホームディレクトリ配下）で実際に使っている設定ファイルと、このリポジトリで管理しているファイルは物理的に別ファイルであり、自動では同期されない。ユーザーが「実際使ってる〇〇をリポジトリに取り込みたい／コピーしたい」と言ったときに使うスキル。

あくまで「ホーム側の実体 → このリポジトリ」への一方向のpullであり、逆方向（リポジトリ→ホーム）のコピーは行わない。

このスキルの範囲はファイルのコピーまで。ブランチ作成・コミット・PR作成は行わない（必要であれば別途 git-commit スキルなどに任せる）。

## パス対応表

このリポジトリ内のパスと、ホーム側の実体パスの対応は以下の通り。

| リポジトリ内パス | ホーム側の実体パス |
|---|---|
| `claude/*` | `~/.claude/*` |
| `codex/*` | `~/.codex/*` |
| `agents/skills/<name>/` | `~/.agents/skills/<name>/`（`~/.claude/skills` はここへのシンボリックリンク） |

上記に当てはまらないファイル（例: `coderabbit/.coderabbit.yaml`）は対応する実体パスが自明でないため、コピー元をユーザーに確認する。

## 手順

1. **pull対象をユーザーに確認する**
   - ユーザーの発話だけで対象ファイルが一意に決まらない場合は、必ず AskUserQuestion で対象を確認する。
   - 候補は上記の対応表に載っているファイル／ディレクトリから提示する（複数選択可）。「全部」ではなく、ユーザーが選んだものだけを対象にする。

2. **実体パスを解決する**
   - 対応表からホーム側の実体パスを解決する。表にない場合は実体パスをユーザーに確認する。

3. **差分を確認する**
   - 対象ごとに `diff` でリポジトリ側とホーム側の差分を確認し、変更点を一言で要約してユーザーに伝える。
   - 差分がない対象はpull不要である旨を伝え、対象から外す。
   - 全対象に差分がなければコピーせず終了する。

4. **pullする（コピーする）**
   - 手順1で選ばれ、かつ差分がある対象についてのみ、ホーム側の実体からリポジトリ側のパスへ `cp`（ディレクトリの場合は `cp -r`）でそのままコピーする。
   - コピーが完了したら、コピーしたファイル一覧をユーザーに報告する。ブランチ作成・コミット・PR作成はここでは行わない。
